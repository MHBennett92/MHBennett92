## FP&A Methodology

### Driver-Based Planning Framework

I build financial models grounded in operational drivers, not static line items. This approach connects business activities directly to financial outcomes.

```mermaid
graph LR
    subgraph Drivers["🔧 Operational Drivers"]
        D1["Volume<br/>Drivers"]
        D2["Headcount"]
        D3["Pricing &<br/>Mix"]
        D4["Channel Mix"]
    end

    subgraph Models["📊 Financial Models"]
        M1["Revenue<br/>Forecast"]
        M2["COGS &<br/>Gross Margin"]
        M3["OpEx<br/>Budget"]
        M4["EBITDA<br/>Projection"]
    end

    subgraph Outputs["📋 Executive Outputs"]
        O1["Board Deck"]
        O2["Variance<br/>Analysis"]
        O3["Scenario<br/>Planning"]
    end

    D1 --> M2
    D2 --> M3
    D3 --> M1
    D4 --> M1
    M1 --> M4
    M2 --> M4
    M3 --> M4
    M4 --> O1
    M4 --> O2
    M4 --> O3

    style Drivers fill:#f8f9fa,stroke:#002856,color:#002856
    style Models fill:#d1ecf1,stroke:#002856,color:#002856
    style Outputs fill:#002856,stroke:#002856,color:#fff
```
